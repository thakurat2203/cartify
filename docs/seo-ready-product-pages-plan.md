# SEO-Ready Product Pages Plan

## Summary

Build the full roadmap version: stable product slugs, server-rendered product pages, legacy ID redirects, dynamic metadata, Open Graph/Twitter metadata, Product JSON-LD, sitemap/robots, and safer configured remote image handling.

The current product detail page is client-rendered at `client/src/app/products/[id]/page.js`, so the main frontend change is to convert it into a server-rendered product route with a small client child for cart/image interactivity.

## Key Changes

- Add SEO product fields to the backend product model:
  - `slug`: generated once, unique, stable, lowercase URL slug.
  - `seoTitle`: optional override, fallback to product name.
  - `seoDescription`: optional override, fallback to product description.
  - `seoKeywords`: optional keyword list.
- Add slug generation and backfill:
  - Generate unique slugs on product creation.
  - Do not change slugs automatically when product names change.
  - Add `server/scripts/backfillProductSlugs.js` and `npm run backfill:product-slugs`.
- Add public product SEO APIs:
  - `GET /api/products/slug/:slug` returns the same product shape as the ID detail API.
  - `GET /api/products/sitemap` returns `{ products: [{ slug, updatedAt }] }`.
  - Keep `GET /api/products/:id` for admin and legacy compatibility.
- Convert product page routing:
  - Rename/replace `client/src/app/products/[id]/page.js` with `client/src/app/products/[slug]/page.js`.
  - If the route param is a Mongo ObjectId, fetch by ID and `permanentRedirect` to `/products/{slug}`.
  - Otherwise fetch by slug and render product content on the server.
  - Move cart button/image fallback state into `product-detail-client.js`.
- Add SEO output:
  - `generateMetadata` with product title, description, canonical URL, Open Graph, and Twitter card data.
  - Product JSON-LD with name, image, description, category, SKU, INR price, canonical URL, and stock availability.
  - `client/src/app/sitemap.js` for root and product URLs.
  - `client/src/app/robots.js` pointing to the sitemap.
- Update storefront links:
  - Catalog and AI result links should use `/products/${product.slug}` with `_id` only as a defensive fallback during migration.
- Add safer image handling:
  - Use `next/image` for product images.
  - Configure allowed image hosts through env, defaulting to `images.unsplash.com`.
  - Existing unsupported image URLs should render the fallback UI instead of breaking the page.

## Test Plan

- Run backend slug backfill locally against existing data:
  - `cd server && npm run backfill:product-slugs`
- Verify API behavior:
  - `GET /api/products?limit=1` includes `slug`.
  - `GET /api/products/slug/{slug}` returns the product.
  - `GET /api/products/sitemap` returns slug/update entries.
  - `GET /api/products/{id}` still works.
- Verify frontend behavior:
  - Product catalog links use slug URLs.
  - `/products/{id}` permanently redirects to `/products/{slug}`.
  - Product page initial HTML contains product name/description.
  - Page source includes canonical metadata and JSON-LD.
  - `/sitemap.xml` includes product URLs.
  - `/robots.txt` references the sitemap.
- Run existing checks:
  - `cd client && npm run lint`
  - `cd client && npm run build`
  - Existing CI Docker builds should pass with the new build args.

## Assumptions

- Implement the full SEO scope, not the metadata-only version.
- Slugs are generated by the system and not manually editable in admin v1.
- Existing product IDs remain valid for API/admin use, but public product URLs become slug-based.
- Currency remains INR because the UI currently displays prices as `Rs.`.
- Allowed product image hosts start with `images.unsplash.com`; more hosts can be added later through env.
